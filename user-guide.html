System Overview

How the three tools work together

The Rack Asset Scanner system is a three-step workflow. You use the Field App on your phone to capture photos and barcode scans on-site, then the Desktop Scanner to process those photos with AI and export to Excel, and finally the Rack Diagram Visualiser to generate a visual front/rear diagram of each rack.

📱 Part 1 — Field App
📱

Field App

Capture rack inventory on-site using your phone

Field App home screen — tap the blue + button to start a new rack survey
1

Open the Field App on your phone

Navigate to karenctz.github.io/rack-scanner/field.html in Chrome (Android) or Safari (iPhone). For quickest access, add it to your home screen:

  • Android Chrome: Tap ⋮ menu → Add to Home screen
  • iPhone Safari: Tap Share → Add to Home Screen
ℹ️

The app works offline once loaded. Your data is saved automatically to the phone — you'll see "💾 Saved offline" flash green at the top after each change.

2

Create a new rack survey

Tap the blue + button in the bottom-right corner. Enter the rack details:

  • Rack # — the rack label (e.g. DC-A-01)
  • Location — room or data hall (e.g. Floor 3, Hall B)
  • Total U — rack height (e.g. 42)

Tap Create. Repeat for each rack you need to document.

3

Add items to a rack

Tap on a rack to open it, then tap + Add Item to log each piece of equipment. For each item:

  • Start U / End U — U position in the rack (app auto-suggests the next free slot based on whether you're working top-down or bottom-up)
  • Category — Server, Network Switch, UPS, Storage, PDU, KVM, or Other
  • Manufacturer / Part Number — fill in what you know
  • Serial Number — tap the scan button to read a barcode, or type manually
  • Part Number — tap the scan button to read a barcode, or type manually
  • Photo — take a photo of the equipment label (used later for AI OCR on the desktop)
  • Notes — any extra notes
💡

U direction is auto-detected. If you start from U42 (top) the app will decrement each new item's suggested U. If you start from U1 (bottom) it increments. Just enter the first two items and the app figures out the pattern.

4

Scan barcodes directly

Both the Serial Number and Part Number fields have a barcode scan button (the barcode icon on the right). Tap either one to open the camera scanner, which reads 1D and 2D barcodes automatically. When a barcode is detected:

  • The correct field fills in automatically
  • The camera closes
  • The scan button turns green to confirm the scan

This is especially useful on equipment that has separate S/N and P/N barcodes on the label — you can scan both without typing anything.

⚠️

Android only: Barcode scanning uses the native Chrome BarcodeDetector API — it works best on Android Chrome. On iPhone you'll need to enter numbers manually or take photos for desktop OCR.

5

Export and share your session

When you've finished logging all racks, go back to the home screen and tap Export All. This creates a .json session file and opens the share sheet so you can:

  • Email it to yourself — tap Mail/Outlook and send to your work email
  • Save to Files / Google Drive
  • AirDrop to a nearby Mac

You'll load this .json file into the Desktop Scanner in the next step.

💡

You can also delete a rack if you made a mistake — scroll to the bottom of any rack page and tap Delete this rack.

🖥️ Part 2 — Desktop Scanner
🖥️

Desktop Scanner

Process photos with AI and export the asset inventory to Excel

Desktop Scanner — paste your Power Automate Flow URL, enter your name, then drop photos or a field session JSON
1

Set up the Desktop Scanner

Open karenctz.github.io/rack-scanner/ on your PC in Chrome or Edge. The toolbar has two fields to fill in before scanning:

  • Power Automate Flow URL — the HTTP trigger URL for your Power Automate flow. Ask your admin if you don't have this.
  • API Key — a shared secret that proves you are authorised to use the flow. Ask your admin for the key. It is masked (shown as dots) and saved automatically so you only need to enter it once per device.
🔑

Keep the API Key private. Do not share it in email or chat. If you think the key has been exposed, ask your admin to rotate it in Power Automate.

2

Load your data

Drag and drop files into the upload zone, or click to browse. You can load:

  • Field session (.json) — exported from the field app. The app will import all rack data immediately.
  • Photos (JPG, PNG, HEIC, WebP, BMP) — individual equipment photos for AI OCR processing. Drop multiple at once.
  • Both together — if your session includes photos, the app auto-splits: items with barcodes/manual data are imported directly; items with photos are queued for OCR.
3

Run AI scanning

Click Scan with AI Builder. The app sends each photo to Microsoft AI Builder (via Power Automate), which reads the text on the equipment labels and extracts:

  • Serial Number
  • Part Number
  • Manufacturer
  • Category (auto-detected from label text)

Results appear in the table row by row as each photo is processed. A progress counter shows how many remain.

⚠️

OCR results are not always perfect. Always review yellow-dot rows (see confidence indicators below) before exporting.

4

Understand confidence indicators

Each row has a coloured dot showing how reliable the data is:

Green — barcode scan (most reliable)
Yellow — AI OCR (review recommended)
Blue — direct import from field app
Grey — manually typed
Purple — OCR result that was manually corrected

Click any cell in the table to edit it. Correcting an OCR field changes it to purple and logs the change to the audit trail.

5

Check for issues before export

The app automatically flags problems as you work:

  • Duplicate serial numbers — an orange banner appears if the same serial number appears in two different rack positions. Click the banner for details.
  • U-gap detection — a panel below the table shows any undocumented U slots per rack (equipment you may have missed).

If you have data across multiple racks, use the rack filter chips at the top of the table to focus on one rack at a time.

6

Export to Excel

Once you're happy with the data, click one of the export buttons:

  • Export to Excel — downloads a .xlsx file with two sheets: Asset Inventory (all rack data) and Audit Log (full action history)
  • Export ZIP (Excel + Photos) — downloads a .zip containing the Excel file plus all the original photos, organised by rack
💡

The export validates your data first. If any rows are missing a Rack # or Serial Number, you'll see a warning asking you to fix them before continuing.

7

Add rows manually

If you need to add items that don't have photos or barcodes:

  • Click + Add Row to insert a blank row
  • Or click the + Add row link at the bottom of the table
  • Fill in the fields directly in the table
📊 Part 3 — Rack Diagram Visualiser
📊

Rack Diagram Visualiser

Generate front & rear visual diagrams from the exported Excel file

Rack Diagram Visualiser — drop your exported Excel file to generate front and rear views of every rack
1

Open the Rack Diagram Visualiser

Open karenctz.github.io/rack-scanner/rack-diagram.html on your PC.

2

Load the Excel file

Drag and drop the rack-assets-YYYY-MM-DD.xlsx file exported from the Desktop Scanner onto the upload zone, or click to browse for it.

The visualiser reads the Asset Inventory sheet and automatically groups equipment by rack.

3

View the rack diagrams

One section appears per rack, showing a front view and a rear view side by side. Each item is drawn at the correct U positions with U1 at the bottom (matching physical rack orientation). Items are colour-coded by AssetType:

Compute Network Backup IP Telephone Other

A colour legend is shown below each rack. Gaps (empty U positions) are shown as dark empty slots so you can immediately see free space.

💡

Hover over any item to see a tooltip with full details: AssetType, U position, Brand, Description, Serialnum, Status, and Department.

4

Filter by rack

If your Excel contains multiple racks, use the rack filter chips at the top to show only specific racks. Click a chip to toggle it on or off. All racks are shown by default.

5

Adjust rack height if needed

Use the Rack Height dropdown (24U / 36U / 42U / 48U) to match your physical racks. You can also change the px per U value to make diagrams taller or shorter on screen.

💡

If items appear clipped or overflow the rack, set Rack Height to a larger value than the highest End U in your data.

6

Print or save as PDF

Click the Print / Save PDF button in the top-right corner. In the print dialog:

  • To save as PDF: set Destination to Save as PDF
  • To print: select your printer as usual

The layout switches to a clean white background for paper. The page URL is suppressed automatically so your PDF contains only the diagram.

💡

Set Layout to Landscape in the print dialog if you have many racks side by side — it fits more on each page.

🛠️

Tips & Troubleshooting

Common questions and how to resolve them

📷 Tips for taking good label photos

  • Hold the camera parallel to the label — avoid angling
  • Ensure the label is fully lit — use your phone torch if the rack interior is dark
  • Capture the whole label in frame, not just the serial number barcode
  • AI Builder reads text best when the image is sharp and in focus — tap to focus before shooting
  • Photo orientation is corrected automatically, so portrait or landscape both work

🔍 Barcode scanner not detecting

  • Barcode scanning requires Android Chrome — it does not work on iPhone or desktop browsers
  • Both Serial Number and Part Number fields have a scan button — tap the one for the field you want to fill
  • Make sure you granted camera permission when the browser asked
  • Hold the barcode steady in the camera frame — the scanner needs about 1–2 seconds to lock on
  • If the scanner opens but never detects, try moving closer or further from the barcode
  • Very small or damaged barcodes may not scan — type the value manually instead

🤖 AI OCR gave wrong results

  • Click directly on the wrong field in the table and type the correction
  • The row will change to a purple dot indicating a manual correction, and the original OCR value is logged to the audit trail
  • If a whole photo is unreadable, delete the row and add it manually using + Add Row

🔁 Duplicate serial number warning

  • An orange banner appears when the same serial number is found in two different rows
  • This usually means one was misread by OCR — check both rows and correct the wrong one
  • Dismiss the banner by clicking ✕ once you've resolved the duplicates

📤 Field app export didn't open email

  • The share sheet requires a mail app to be installed on your phone
  • If tapping Export All asks you to save a file instead of showing the share sheet, your browser may not support the Web Share API — save the file to Files/Drive and attach it manually to an email
  • If you tap Cancel on the share sheet, the file is still available in your Downloads folder

💾 Field app data seems lost after closing the browser

  • Data is saved to your phone's local storage — it persists as long as you don't clear your browser data
  • Do not use Private/Incognito mode — data is not saved in private browsing
  • Always export your session before clearing browser data or switching phones

📊 Rack diagram is missing some equipment

  • The diagram reads from the Asset Inventory sheet in the Excel file — make sure you're loading the correct export
  • Items without a Rack # or valid Start U / End U values are not drawn
  • Check that the U values in the spreadsheet are numbers, not text

How It Works

The logic and design decisions behind the app

📦 Data flow between the three apps

The three tools are deliberately separate so that field work and office processing are decoupled — engineers don't need internet on site, and the desktop processing doesn't need the phone nearby.

1. Field App → JSON file

Everything captured on-site (rack metadata, U positions, serial numbers scanned by barcode, part numbers, photos) is stored in the phone's browser localStorage. When you export, it serialises all sessions into a single .json file. Photos are embedded as base64 strings inside the JSON so the file is entirely self-contained — no separate attachment needed.

2. Desktop Scanner → Excel file

The desktop app reads the JSON and splits items into two groups: those that already have serial numbers (barcode-scanned or manually typed) are imported directly without touching the AI. Items that have photos but no serial number are queued for OCR. This avoids unnecessary API calls and keeps costs down.

3. Excel → Rack Diagram

The rack diagram reads the exported .xlsx directly in the browser using SheetJS — no server upload required. It groups rows by Rack # and maps each item's Start U / End U to a pixel position in the SVG diagram.

🤖 How AI OCR works

When you click Scan with AI Builder, each photo is sent as raw binary to a Microsoft Power Automate flow. The flow passes the image to AI Builder Text Recognizer, which reads every character printed on the label and returns the full extracted text.

The desktop app then parses that raw text using pattern matching to pull out:

  • Serial Number — looks for patterns like S/N:, SN:, Serial: followed by an alphanumeric string
  • Part Number — looks for P/N:, PN:, Part No: patterns
  • Manufacturer — matched against a known list (HP, Dell, Cisco, etc.)
  • Category — inferred from keywords in the OCR text (e.g. "Switch" → Network Switch, "UPS" → UPS)

Photos are resized to a maximum of 1600px before sending and EXIF orientation is corrected automatically — so portrait phone photos arrive at the AI in the right orientation regardless of how you held the camera.

🎯 Confidence indicator logic

Every row is assigned a confidence level based on how its data was captured. This helps you know which rows need a human check before exporting.

Barcode

Serial or part number was read by the camera's BarcodeDetector API. The raw value comes directly from the barcode — no interpretation, no guessing. Highest reliability.

Direct import

All fields were typed manually in the field app and imported as-is — no AI involved. Reliable if the engineer entered the data correctly.

OCR – review

Data was extracted from a photo by AI Builder. The AI is accurate but not perfect — blurry photos, unusual label formats, or similar characters (0/O, 1/I) can cause mistakes. Always spot-check yellow rows.

Manual entry

A field was typed directly into the desktop table. No AI or scanner involved.

AI corrected

An OCR result was edited manually after the fact. The original AI value is logged in the audit trail so you can always see what the AI read vs. what was corrected.

📷 How barcode scanning works

The field app uses the browser's built-in BarcodeDetector API (available on Android Chrome) — no external library or internet connection needed. The camera feed is sampled every 150ms and each frame is passed to the detector. When a barcode is found, the camera stops immediately and the value is filled into the field that triggered the scan (serial number or part number).

Supported barcode formats: Code 128, Code 39, QR Code, Data Matrix, EAN-13, EAN-8, UPC-A, UPC-E, ITF, PDF417, Aztec.

On devices without BarcodeDetector (iPhone, desktop), the app falls back to the ZXing library loaded on-demand, or you can take a photo of the label instead and let the desktop OCR read it.

🔢 U position auto-detection

Engineers may work a rack top-down (starting at U42) or bottom-up (starting at U1). The field app detects the direction automatically so the suggested next U is always correct:

  • After two or more items are logged, the app compares the first item's U with the last item's U. If U is decreasing → top-down mode; if increasing → bottom-up mode.
  • With only one item, it assumes top-down if that item is above U21, bottom-up if at U21 or below.

The suggested next U is always one slot past the outermost item already logged — so if you just logged U38–U40, the next suggestion is U37 (top-down) or U41 (bottom-up).

🔍 Duplicate & U-gap detection

Duplicate detection runs automatically every time the table re-renders. It groups all rows by serial number and flags any serial number that appears in more than one row. An orange warning banner shows the duplicates so you can correct the wrong entry before exporting.

U-gap detection groups rows by Rack # and checks whether the U positions in use cover the full rack continuously. Any contiguous block of U slots with no logged equipment is reported as a gap — meaning either that space is genuinely empty or an item was missed. The gap panel is collapsible and shown below the main table.

📋 Audit trail

Every action taken in the desktop scanner is logged to an in-memory audit trail. Each entry records:

  • Timestamp — when the action happened
  • Engineer — the name entered in the Engineer field
  • Action — what happened (OCR scan, manual edit, import, barcode scan)
  • Rack / Unit / Serial — which asset was affected
  • Old value — for corrections, what the original OCR read before the edit

When you export to Excel, the audit trail appears as a second sheet called Audit Log in the same .xlsx file — so the full history of who reviewed what travels with the asset data.

💾 Offline storage in the field app

The field app stores all data in browser localStorage — a per-origin key-value store built into every browser. There is no server, no account, and no sync. Data persists across sessions as long as you don't clear browser storage.

Every time you save an item, the entire sessions array is serialised to JSON and written to localStorage. The "💾 Saved offline" indicator flashes green to confirm each write succeeded.

Photos are stored as compressed JPEG base64 strings (max 1600px, ~82% quality). A rack survey with 20 items and 2 photos each typically occupies 10–20 MB of localStorage. Most browsers allow 5–10 MB per origin; if storage fills up, the app will show an error and you should export and clear old sessions.

🔑 Why an API Key is required

The Power Automate HTTP trigger URL is a plain HTTPS endpoint with no built-in authentication. Anyone who obtains the URL could send requests to it — consuming your organisation's AI Builder quota and potentially processing unauthorised data.

The API Key adds a layer of protection. Every request the desktop app sends includes an x-api-key HTTP header containing the secret. Power Automate checks this header at the start of the flow — if it is missing or wrong, the flow immediately returns 401 Unauthorized and no AI processing happens.

How it flows:
1. Engineer enters the API Key in the desktop app — saved to localStorage, never transmitted until a scan is triggered.
2. On scan, the app POSTs the photo binary with headers: Content-Type: image/jpeg and x-api-key: <secret>.
3. Power Automate checks triggerOutputs()['headers']['x-api-key'] equals the expected value.
4. Match → AI Builder processes the image. No match → flow returns 401, desktop app shows an error.

For admins — setting up the key in Power Automate:

  • Add a Condition action as the first step after the HTTP trigger
  • Left side: triggerOutputs()['headers']['x-api-key']
  • Operator: is equal to
  • Right side: your chosen secret (e.g. rack-scanner-k9x2mq)
  • If Yes → continue to AI Builder steps
  • If No → add a Response action, status 401, body Unauthorized
⚠️

The API Key is stored in browser localStorage in plaintext. Use only on managed, encrypted devices. If the key is compromised, change it in Power Automate and redistribute the new key to engineers.
